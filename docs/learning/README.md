# Beginner Learning Guide

This folder is for learning the project from zero. It is intentionally more
beginner-friendly than the HLD, LLD, and code walkthroughs.

Read the files in this order:

1. [00-overall-flow.md](00-overall-flow.md)
   - What the project does.
   - What happens from user click to database query to UI result.
   - How the backend, frontend, databases, graph, LLM, and Docker fit together.

2. [01-step-by-step-learning-plan.md](01-step-by-step-learning-plan.md)
   - The full learning roadmap.
   - What concepts to learn before each module.
   - Which files to study.
   - Checkpoints and small exercises.

3. [02-glossary.md](02-glossary.md)
   - Project terms, warehouse terms, and tech-stack terms in plain English.

How to use this folder:

- Do not start by reading random code files.
- First understand the problem and the data flow.
- Then study one module at a time.
- After each module, answer the checkpoint questions before moving ahead.
- When a file feels confusing, come back to the glossary and the overall flow.

The goal is not just to know what the files are. The goal is to become able to
explain, debug, modify, and defend the project.
